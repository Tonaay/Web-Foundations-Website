Futher Reading – Information Security Management Principles (Second Edition)

Notes on Further Reading

Confidentiality - Making sure that information is not made available or disclosed to unauthorised personell, entities or processes.

Integrity - Information is only useful if it is complete and accurate, and remains so. This is important to make sure that certain people have appropriate authority to alter, update and delete information.

Availability - Making sure that information is always available, therefore having to decide between security or availablity of information.

Assets = Anything that has value to an organisation. Assets are split into three types.

  1. Pure information
  2. Physical assets such as computer systems or buildings.
  3. Software which is used to manage and process information.

Impact - The result of an information security incident, caused by a threat which affects the assets of the organisation.

Information security policy concepts

When an organisation forms a policy for its management of information assurance. This is usually a short statement from a chief executive where risks are acknowledged and that the business will take appropriate measures to deal with them.

Processing Information

Social media services will categorise information in terms of how sensitive/confidential it is and if it would be appropriate to display to the public. They might receive the user's email and phone number, in which they won’t want it displayed to others on the site. However their name might be suitable to display publicly so other users can find them. It is critical to point out however, that a user’s password should not be displayed publicly to prevent unauthorised access or fraud. When exchanging information to third parties, it is essential for the service to gain authorisation to do this, with clear expression of what exact information can be shared.

Graphics with SVG

This is a drawing of my friend I did using illustrator. I exported the file as an SVG and imported it into this page. You can test whether the quality has changed by zooming in.

SVG is a good replacement instead of using JPEG or PNG because it doesn't matter about your screen size, resolution, as well as zoom levels. SVG is vector based meaning that it is built from paths, shapes and lines. This allows the size to not lose it's quality at any resolution which is good for digital artworks, fonts, or even music sheets etc.

SVG's are just code which means loading is much faster as the file size is a lot smaller than that of a PNG or JPEG. As it is just code, you can manipulate certain lines and change parts of a drawing or even make them animate.

However, there are downsides of using SVG's such as that old browsers may not support them and also, complicated artwork or images which have thousands of nodes can be complex to implement so it is better to use JPEG or PNG in these circumstances.

SVG's are best used for small logos, icons, and simple graphics.

Databases in Websites

Websites are usually running on computer clients and on mobile computers.

Databases are usually stored inside a computer also. This database may be written in SQL therefore, needs to be translated to HTTP so they can communicate with each other, the computer can talk SQL to the database and HTTP to the clients using the website.

This is also known as Three-tier Architecture which includes Business Logic, Security and Validation as well as Data Storage Security.

In modern websites, there is an API between clients and the app server. API is a set of functions and procedures that allow a creation of an applications which can access features or data from the website.

In one of the lecturer's web application, I was able to play around with a shop page where I could add a list of goods into a cart and then check them out. This would then produce a list of all the items I had in API code. This would show the order containing the information of the buyer, as well as all the product quantity, price and other details.
